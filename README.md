## Сборка и зависимости

Проект использует CMake для управления сборкой и зависит от:
- Библиотек WinAPI (автоматически доступны в Windows)
- GDI32 для графических операций
- User32 для создания интерфейса

## Ключевые алгоритмические решения

1. **Проекция из 5D в 2D**: Используется ортогональная проекция с помощью матрицы камеры 5×2
2. **Вращение в n-мерном пространстве**: Реализовано через последовательность элементарных вращений в плоскостях
3. **Генерация гиперкуба**: Вершины генерируются как битовые маски, рёбра соединяют вершины с расстоянием Хэмминга 1
4. **Интерполяция вращения**: Углы вращения накапливаются и применяются к матрице камеры

Этот подход сочетает математическую строгость многомерной геометрии с практическими аспектами компьютерной графики, позволяя визуализировать объекты, которые невозможно непосредственно наблюдать в трёхмерном пространстве.

## Сборка и запуск

### Требования
- Компилятор с поддержкой C++11
- CMake версии 3.10 или выше
- Windows OS

### Инструкция по сборке
```bash
# Создание директории для сборки
mkdir build
cd build

# Генерация проекта CMake
cmake ..

# Сборка проекта
cmake --build .

# Запуск приложения
./MyGraphProject
```

## Управление

### Активация управления
- Нажмите левую кнопку мыши в окне приложения для активации режима управления

### Управление вращением
- Клавиши 0-9 (основные или на NumPad): вращение вокруг различных плоскостей (каждая клавиша увеличивает соответствующий угол на 15°)

### Управление положением камеры
- Клавиши A, S, D, F, G: перемещение камеры в отрицательном направлении по осям X, Y, Z, W, V соответственно
- Клавиши Q, W, E, R, T: перемещение камеры в положительном направлении по осям X, Y, Z, W, V соответственно
